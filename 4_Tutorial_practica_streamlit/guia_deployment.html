01
💾 Guardar el modelo entrenado
Exportar el pipeline de GridSearchCV a un archivo .joblib desde VS Code
Por qué es necesario este paso

El modelo entrenado vive en memoria mientras el notebook está abierto. Para usarlo en la app web hay que serializarlo a disco con joblib, generando un archivo .joblib que la app puede cargar sin necesidad de reentrenar.

Agrega esta celda al final de tu notebook
1️⃣
Abre el notebook en VS Code
Abre el archivo 3_Solucion_microproyecto_dos.ipynb y asegúrate de que todas las celdas anteriores (incluyendo el GridSearchCV) ya están ejecutadas.
2️⃣
Crea una nueva celda al final del notebook
Haz clic en el + Code que aparece al pasar el mouse debajo de la última celda.
3️⃣
Pega y ejecuta este código
📓 nueva celda en el notebook 
import joblib
import os

# Crear carpeta resources si no existe
os.makedirs("resources", exist_ok=True)

# Guardar el mejor modelo encontrado por GridSearchCV
joblib.dump(grid_search.best_estimator_, "resources/model.joblib")

print("✅ Modelo guardado en: resources/model.joblib")
print(f"   Parámetros: {grid_search.best_params_}")
print(f"   F1-Score:   {grid_search.best_score_:.4f}")
4️⃣
Verifica que se creó el archivo
Después de ejecutar, deberías ver en el panel izquierdo de VS Code una nueva carpeta resources/ con el archivo model.joblib adentro.
⚠️
Si ves el error "name 'grid_search' is not defined", significa que el GridSearchCV no se ejecutó todavía. Ejecuta todas las celdas del notebook en orden (Menú → Run → Run All) y luego vuelve a ejecutar esta celda.
02
📁 Organizar los archivos del proyecto
Crear la estructura de carpetas correcta en VS Code
Estructura que debe quedar en tu computador
📁 ODS_Classifier/ ← carpeta raíz del proyecto
  📄 streamlit_app.py     ← app principal ✅ ya tienes este archivo
  📄 requirements.txt      ← dependencias ✅ ya tienes este archivo
  📄 save_model.py        ← (solo referencia, no se sube)
  📁 src/
    📄 DataPreprocessing.py  ← ✅ ya tienes este archivo
    📄 ModelController.py    ← ✅ ya tienes este archivo
  📁 resources/
    📄 model.joblib         ← 🆕 el que acabas de generar
Cómo organizar desde VS Code
1️⃣
Crea una carpeta llamada ODS_Classifier en tu computador
Puede ser en tu escritorio o en Documentos. Esta será la carpeta raíz del proyecto.
2️⃣
Abre esa carpeta en VS Code
Menú File → Open Folder → selecciona la carpeta ODS_Classifier.
3️⃣
Copia los archivos que ya tienes descargados
En el panel izquierdo (Explorer), coloca los archivos así:
• streamlit_app.py → directamente en ODS_Classifier/
• requirements.txt → directamente en ODS_Classifier/
• DataPreprocessing.py y ModelController.py → dentro de una subcarpeta llamada src/
4️⃣
Copia el model.joblib que generaste
El archivo model.joblib que apareció en la carpeta resources/ al ejecutar la celda del paso anterior → cópialo a ODS_Classifier/resources/model.joblib.
💡
Puedes crear subcarpetas desde VS Code haciendo clic derecho en el Explorer → New Folder.
03
🐙 Subir el proyecto a GitHub
Crear un repositorio y publicar todos los archivos
Opción A — Desde la interfaz web de GitHub (más fácil)
1️⃣
Crea un repositorio nuevo
Ve a github.com → botón verde "New" (esquina superior izquierda) → ponle nombre, ej: clasificador-ods → marca "Public" → clic en "Create repository".
2️⃣
Sube los archivos arrastrando
En la página del repositorio recién creado verás un enlace "uploading an existing file". Haz clic ahí.

Arrastra todos los archivos y carpetas (streamlit_app.py, requirements.txt, la carpeta src/, la carpeta resources/) a la zona de upload.
3️⃣
Confirma el commit
Al fondo de la página escribe un mensaje como "Agregar clasificador ODS" y haz clic en "Commit changes".
Opción B — Desde la terminal de VS Code (con Git)
⌨️
Abre la terminal en VS Code
Menú Terminal → New Terminal. Ejecuta estos comandos en orden:
💻 Terminal de VS Code 
# Inicializar repositorio Git en la carpeta
git init
git add .
git commit -m "Primer commit: clasificador ODS"
git branch -M main

# Conectar con GitHub (reemplaza con tu usuario y nombre de repo)
git remote add origin https://github.com/TU_USUARIO/clasificador-ods.git
git push -u origin main
⚠️
El model.joblib puede ser grande. Si pesa más de 100 MB, GitHub lo rechazará. En ese caso usa Git LFS o reduce el modelo con menos componentes SVD. En la mayoría de los casos con este dataset el archivo pesa entre 5–30 MB y no hay problema.
✅
Al final debes ver en tu repositorio de GitHub los archivos: streamlit_app.py, requirements.txt, carpeta src/ con los dos módulos, y carpeta resources/ con el model.joblib.
04
🚀 Desplegar en Streamlit Cloud
Conectar el repositorio de GitHub y publicar la app
Pasos en Streamlit Cloud
1️⃣
Inicia sesión en Streamlit Cloud
Ve a share.streamlit.io → inicia sesión con tu cuenta de GitHub (esto conecta automáticamente tus repositorios).
2️⃣
Crea una nueva app
Haz clic en el botón "New app" (esquina superior derecha).
3️⃣
Completa el formulario de deployment
Llena los campos exactamente así:
📋 Formulario de Streamlit Cloud
Repository:   TU_USUARIO/clasificador-ods
Branch:       main
Main file:    streamlit_app.py
4️⃣
Haz clic en "Deploy!"
Streamlit Cloud clonará tu repositorio, instalará las dependencias del requirements.txt y ejecutará la app. Este proceso tarda entre 2 a 5 minutos.
5️⃣
Obtén tu URL pública
Una vez desplegada, recibirás una URL del tipo:
https://TU_USUARIO-clasificador-ods-streamlit-app.streamlit.app

¡Esa URL es la que compartes con el profesor!
💡
Si la app muestra un error de "ModuleNotFoundError" revisa que todos los paquetes del requirements.txt estén escritos correctamente. Si muestra "FileNotFoundError: model.joblib" verifica que el archivo sí quedó en la carpeta resources/ del repositorio.
✓
🎉 ¡Deployment completado!
